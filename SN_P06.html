Introduccion

MODELO DIFUSO TSUKAMOTO
Se caracteriza por presentar el consecuente de cada regla borrosa mediante un conjunto borroso con una función de pertenencia monótona.
La salida inferida por cada regla es un valor numérico inducido por el grado de activación de la regla. La salida global se toma como la media ponderada de todas las salidas parciales (PROMEDIO). Este tipo de modelos permite eliminar el consumo de tiempo correspondiente a la desborrosificación, sin embargo, no son modelos muy utilizados por no ser tan transparentes como los de Mamdanio o Sugeno.
El mecanismo de inferencia no sigue estrictamente la regla composicional de inferencia y la salida siempre es numérica, incluso cuando las salidas son borrosas.
En resumen tenemos que el modelo difuso de TSUKAMOTO
En este modelo la función consecuente es un set difuso con una función monotónica
Si X es A, Y es B entonces Z es C
La salida de cada regla es un valor exacto inducido por la fuerza de disparo de cada regla y cada regla tiene una salida EXACTA
Este método no necesita la defuzificación ya que arroja las salidas exactas para cada regla usando el promedio

3.12 Reporte el diagrama de flujo del GA utilizado.
El proceso de AG comienza utilizando el conjunto inicial de soluciones, denominado población inicial, que se genera aleatoriamente y se codifica posteriormente en cromosomas binarios. El FLC se actualiza con cada cromosoma y, en consecuencia, se evalúa utilizando una función de aptitud adecuada. En este estudio, hemos seleccionado el error cuadrático medio (RMSE) como una función de aptitud
Por lo tanto, la mejor solución es un vector que alcance el valor mínimo de la función RMSE. De hecho, la salida predicha (oi) se compara con su objetivo correspondiente (ti), para evaluar la función RMSE. Posteriormente, para crear la diversidad de la población, se aplican operadores de selección, cruce y mutación y se evalúa una nueva población. Este proceso continúa durante varias iteraciones hasta que se alcanzan los criterios de parada (el número máximo de generación, el número máximo de iteraciones, etc.). Al final, la solución con el RMSE más bajo se selecciona como la mejor y se devuelven sus parámetros optimizados.
